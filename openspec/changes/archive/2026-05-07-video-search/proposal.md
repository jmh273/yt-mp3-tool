# 影片搜尋功能 (Video Search)

## 需求描述
使用者希望能夠在應用程式中透過「關鍵字」搜尋 YouTube 影片，以發掘更多非訂閱頻道的內容，並可直接加入下載佇列。

## 實作方案評估

### 方案 A：使用官方 YouTube Data API (Search Endpoint)
- **作法**：透過 `youtube.search().list(part="snippet", q=keyword, type="video", maxResults=50)` 進行搜尋。
- **優點**：實作簡單，與現有的 API 架構完美整合，搜尋速度極快且精確。
- **缺點**：**配額消耗極大**！根據官方文件，每次 Search API 請求會消耗 **100 Quota**。若使用者頻繁搜尋，很容易耗盡每日 10,000 Quota 的限制。

### 方案 B：使用 `yt-dlp` 的 `ytsearch` 功能 (免配額)
- **作法**：後端利用專案已內建的 `yt_dlp` 模組，執行類似 `ytsearch50:{keyword}` 的指令來爬取搜尋結果。
- **優點**：**0 API Quota 消耗**，完全免費。
- **缺點**：
  - 搜尋速度較官方 API 稍慢 (大約需多花 2-5 秒)。
  - 需要自訂解析 `yt_dlp` 回傳的格式，將其映射為前端所需的欄位結構。

### 方案 C：混合模式 (官方 API 搭配快取)
- **作法**：使用官方 API，但在後端加上 Redis 或本機字典快取 (以關鍵字為 Key)。
- **缺點**：搜尋關鍵字變異太大，快取命中率通常很低，最終依然會面臨 Quota 消耗過重的問題。

## 推薦方案
強烈建議採用 **方案 B (`yt-dlp` 免配額搜尋)**。
由於 YT-MP3 專案主打重度使用者且已遭遇過多次 Quota 限制的痛點，為了確保核心功能 (如訂閱更新) 正常運作，應盡可能避免在輔助探索功能上浪費每次 100 點的 Quota。
