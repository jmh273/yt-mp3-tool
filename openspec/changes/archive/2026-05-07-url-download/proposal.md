# 網址下載功能 (URL Download)

## 需求描述
使用者希望可以直接貼上 YouTube 網址（單一影片或播放清單），系統解析後會列出該影片的縮圖、標題等資訊，接著可以勾選並加入下載佇列。這為非訂閱、非搜尋的額外影片提供了彈性的下載管道。

## 實作方案評估

### 方案 A：手動正則解析 + 官方 YouTube Data API
- **作法**：撰寫正規表示式取出網址中的 `v=` (Video ID) 或 `list=` (Playlist ID)，再打 API 去查。
- **優點**：查詢速度極快。
- **缺點**：
  - 需自行處理極多種網址變體 (`youtube.com`, `youtu.be`, 帶 timestamp 的網址等)。
  - **需要消耗 API Quota**，違背我們盡量節省配額的原則。

### 方案 B：使用 `yt-dlp` 的 URL 解析功能 (強烈推薦)
- **作法**：建立新的後端端點 `/url-preview?url=...`，透過 `yt_dlp.extract_info(url, extract_flat=True)` 處理。
- **優點**：
  - **完全不消耗 API Quota**。
  - `yt-dlp` 是專業的解析器，無論使用者貼的是標準網址、短網址 (`youtu.be`)，甚至是**整個播放清單 (Playlist)** 的網址，它都能完美解析並提取資訊。
  - 如果使用者貼上的是播放清單，系統可以直接回傳整個清單的所有影片讓使用者勾選。
- **缺點**：遇到包含幾百部影片的龐大播放清單時，解析可能需要等候 3~5 秒。

## 預期 UI / UX 變更
1. **導覽列 (`HomeView.vue`)**：左側側邊欄新增「🔗 網址下載」按鈕。
2. **網址解析組件 (`UrlDownloadFeed.vue`)**：
   - 頂部：提供一個輸入框與「解析」按鈕（支援 Enter 鍵）。
   - 解析完成後：在下方以標準網格 (Grid) 顯示該影片的資訊（縮圖、標題、長度）。
   - **輔助功能**：為了應對「播放清單」的場景，會貼心提供一個「全選 / 全不選」的按鈕，方便整批加入下載。單一影片解析出來時則預設自動勾選。
   - 所有勾選狀態皆無縫綁定至現有的右側下載佇列 `downloadStore`。
